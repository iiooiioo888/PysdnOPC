本模块由两个独立入口组成：
- `cli_collab.py` 实现 `opc-collab` 命令行（通过 `argparse` 动态为每个合作工具生成子命令），统一从 `--args-json-file` / `--args-stdin` / `--args-json` / `--set` 收集参数，再经 `_dispatch` 在运行时选择走 `layer4_tools.collaboration_rpc`（Windows 外部代理 RPC）或 `layer4_tools.collaboration_dispatch`（本地同步分发）。
- `mcp_client.py` 定义抽象基类 `_MCPConnectionBase` 及两个具体连接 `MCPServerConnection`（stdio 本地进程）和 `MCPRemoteConnection`（StreamableHTTP 优先、SSE 回退），并通过 `MCPManager` 将发现的工具包装成 `ToolDefinition` 注入 OPC ToolRegistry，工具名以 `server_prefix_` 前缀避免冲突。
- `__init__.py` 仅做包级副作用：在导入时调用 `sanitize_windows_sslkeylogfile()` 清理 `SSLKEYLOGFILE`，防止 Windows 上 OpenSSL 崩溃，并导出 `__version__`。
依赖方向单向向外：CLI → layer4_tools；MCP 客户端 → `opc.layer4_tools.registry.ToolDefinition`；两者均不反向依赖上层。
