- 每个 Skill 文件顶部使用 YAML Front Matter 声明 name/description/domain/trigger/always_on 五个必填字段，作为统一的 Skill 注册契约。
- 通过 `domain` 数组将 Skill 标注到多个领域标签，而非用文件名或子目录区分领域。
- 正文采用「## Workflow / ## Best Practices / ## Security」等固定二级标题组织内容，使提示词结构一致、易于解析。
- 需要常驻生效的 Skill 显式设置 `always_on: true`，普通 Skill 保持 `always_on: false` 并通过 `trigger` 条件匹配激活。
