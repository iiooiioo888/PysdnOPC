本模块是一个纯数据/提示词目录，不包含可执行代码；每个 `.md` 文件即一个独立的 Skill 定义，由上层 OPC 运行时在任务触发时按需加载。
- 统一的前置 YAML Front Matter 元数据块（`name`、`description`、`domain`、`trigger`、`always_on`）作为 Skill 的注册契约，供解析器读取以决定何时激活以及归属哪些领域。
- `external_agents.md` 标记 `always_on: true`，作为常驻决策引导，用于在原生 Agent 与外部 CLI Agent（Cursor / Claude Code / Codex）之间做路由选择；其余 Skill 默认 `always_on: false`，仅在匹配的 `trigger` 文本出现时激活。
- `domain` 字段将 Skill 映射到项目级领域标签（coding、frontend、backend、devops、writing、documentation、automation、general），形成“Skill → domain”的多对多关系，便于按上下文过滤。
- 依赖方向单向：Skill 仅被上层运行时消费，不反向引用任何 Python 包或配置文件。
