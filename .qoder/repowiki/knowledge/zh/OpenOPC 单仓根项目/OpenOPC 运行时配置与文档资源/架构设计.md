本模块为纯数据/文档型叶子目录，不含可执行代码，由上层 Python 运行时通过路径解析加载：
- config/ 提供五类 YAML 配置模板，被 `opc init` 复制至用户 `.opc/config/` 后生效：`system_config.yaml`（系统开关、权限策略、MCP servers、sandbox、prompt harness 等核心运行时参数）、`llm_config.yaml`（默认模型、api_base、上下文窗口覆盖）、`agent_config.yaml`（外部 agent 如 claude/codex/cursor/opencode 的命令与 session 标志，以及 native_subagents general/explore/plan/implement/verify 的行为）。
- `channel_config.yaml` 声明所有内置 channel 桥接器（telegram、whatsapp、discord、feishu、mochat、dingtalk、email、slack、qq、matrix）的启用开关与凭据占位；`company_corporate_config.yaml` 定义公司拓扑、员工角色模板与 fallback 占位，配合 `docs/company-metadata-ownership.md` 说明企业元数据所有权规则。
- docs/ 包含通道桥接指南、CLI slash 命令参考、架构图示与 assets 图片，作为用户手册与开发者参考。
- scripts/reset_stuck_task.py 是运维辅助脚本，用于重置卡住的运行任务。
- .opc/ 是 `opc init` 生成的工作区骨架（config/prompts/skills），实际内容在初始化时写入，当前为空目录仅作约定式占位。
依赖方向单向：上层 runtime 读取这些 YAML 并注入到 AgentRuntime / ChannelManager / CompanyTopology 等组件，本模块不反向依赖任何 Python 包。
