### Company Mode
- 定义：OpenOPC 的一种执行模式，将一个任务简报编译为运行期会话 + 多个角色拥有的工作项（work item），每个角色在其自身会话中并行执行，通过审查、委派、最终决策者等机制协作推进。
- 别名：公司模式、company

### Task Mode
- 定义：OpenOPC 的单 Agent 执行模式，类似 LobeChat 的单对话工作区，可选择 Native/Codex/Claude Code/Cursor/OpenCode 作为执行代理，适合一次性任务。
- 别名：任务模式、task

### Work Item
- 定义：Company Mode 中由某个角色拥有的最小可执行单元，具有状态机驱动的阶段（规划/执行/审查/阻塞/完成），决定看板列、负责人和可运行性。
- 别名：工作项、work_item

### Kanban
- 定义：OpenOPC 的任务看板视图，在 Task Mode 下是项目级看板，在 Company Mode 下跟随当前运行期会话展示工作项及其状态流转。
- 别名：看板、kanban board

### Office UI
- 定义：OpenOPC 的 Web 前端界面，包含 Workspace（主工作区）、Office（可视化办公室地图）、Org（组织架构编辑）三个主要页面，用于监控和管理 AI 公司员工。
- 别名：办公界面、office

### Talent Template
- 定义：可导入的员工模板，描述角色的职责、技能和经验，可从 talent library 批量导入并通过 `opc talent hire` 雇佣到空缺角色。
- 别名：人才模板、talent template、agency-agents

### .opcpkg
- 定义：OpenOPC 的组织架构打包格式，包含 roles、work_item_templates、gates、prompts、skills 等内容，支持导出/安装/分享整个公司架构。
- 别名：opcpkg、package

### Channel Provider
- 定义：OpenOPC 的外部消息通道适配器，将不同平台（飞书、Telegram、Discord、Slack 等）的消息标准化为 UserMessage/SystemMessage 并路由到引擎。
- 别名：通道提供者、channel

### Bridge Provider
- 定义：需要独立桥接服务的通道类型（WhatsApp、Mochat），OpenOPC 仅通过 WebSocket/HTTP 与外部桥服务通信，不直接管理桥进程。
- 别名：桥接通道、bridge

### External Agent
- 定义：OpenOPC 可调用的外部 CLI 代理（Codex、Claude Code、Cursor、OpenCode），通过命令行交互执行任务，支持会话复用和审批模式。
- 别名：外部代理、external agent

### Native Runtime
- 定义：OpenOPC 内置的执行引擎，支持流式 LLM 调用、子代理、工具钩子、验证门控、微压缩、预取等高级特性，是 Task Mode 的默认执行器。
- 别名：原生运行时、native runtime

### Autonomy Policy
- 定义：OpenOPC 的自主性策略，通过风险等级（low/medium/high/critical）、敏感关键词、安全命令前缀、工具首次使用审批等机制控制 agent 的自动执行范围。
- 别名：自主策略、autonomy

### OPC Home
- 定义：OpenOPC 的本地数据目录（默认 `.opc/`），包含配置、全局/项目记忆、项目运行时元数据、UI 状态等，可通过 `OPC_HOME` 环境变量自定义路径。
- 别名：OPC 家目录、opc_home

### Secretary
- 定义：OpenOPC 的系统记忆管家，负责维护 OPC 系统记忆、分析比较项目、引导 YAML 配置设置，未来将增强为更强的配置和记忆管理者。
- 别名：秘书、secretary

### Market Preset
- 定义：OpenOPC 市场中的预置组织架构图，可浏览、预览、应用到当前组织，支持导出/安装社区分享的架构包。
- 别名：市场预设、preset、architecture preset
